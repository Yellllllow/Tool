一些常见问题
每次写完文件都要命令行编译一次？

我们可以让源文件实时同步，自动编译。  命令如下：
sass --watch index.scss:index.css

如果有很多文件，难道每个css都需要这样写？

我们不仅可以让某个文件，自动编译。还可以让整个文件夹的文件都自动编译。 命令如下：
sass --watch D:\hbuild_workspace\Sass\css:D:\hbuild_workspace\Sass\css

红色文件夹是sass文件目录:蓝色文件夹为css文件存放目录
此时sass会实时监测文件的改变并自动编译，但如果关闭命令行，监听就失效了

有没有更好的解决方案？

Hbuilder中提供了一个插件，我们只需要将sass环境导入其中，就可以自动编译了。

混合(mixin)

@mixin border-radius($size){
    -moz-border-radius: $size;
    -webkit-border-radius: $size;
    border-radius:$size;
}
//引入
#box {
    @include border-radius(10px); 
}
//生成
#box {
    -moz-border-radius: 10px;
    -webkit-border-radius: 10px;
    border-radius:10px;
}
//之所以叫混合，是因为它的本质还是代码片段的合并，不过跟继承比起来，它灵活的地方在于可以使用参数
//对于浏览器兼容的封装提供了极大的便利

问题： sass中不支持中文注释，如何是好？
找到ruby的安装目录，里面也有sass模块，类似这样样的路径：
C:\Ruby\lib\ruby\gems\2.3.0\gems\sass-3.4.22\lib\sass
在这里面有个文件叫engine.rb,添加一行代码
Encoding.default_external = Encoding.find('utf-8')
放在所有的require xxxxxx 之后即可。
